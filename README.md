# Magic Recipe

An AI-powered recipe generator app built with Flutter and Serverpod. This application allows users to input ingredients and receive personalized recipes generated by Google's Gemini AI. Recipes are created in Uzbek language and stored in a database for future reference.

## Features

- **AI Recipe Generation**: Input ingredients and get creative recipes powered by Gemini AI
- **Recipe History**: Save and view previously generated recipes
- **User Authentication**: Secure login system using Serverpod Auth
- **Cross-Platform**: Available as a Flutter mobile app and web application
- **Database Storage**: Recipes stored in PostgreSQL with vector support
- **Real-time Updates**: Seamless synchronization between client and server

## Tech Stack

- **Frontend**: Flutter (Mobile & Web)
- **Backend**: Serverpod (Dart)
- **Database**: PostgreSQL with pgvector extension
- **Cache**: Redis
- **AI**: Google Gemini AI
- **Authentication**: Serverpod Auth with email/password

## Prerequisites

Before running this project, ensure you have the following installed:

- [Dart SDK](https://dart.dev/get-dart) (version 3.8.0 or higher)
- [Flutter SDK](https://flutter.dev/docs/get-started/install) (version 3.32.0 or higher)
- [Docker](https://www.docker.com/get-started) and Docker Compose
- A Google Gemini API key

## Installation

1. **Clone the repository:**
   ```bash
   git clone <repository-url>
   cd magic_recipe
   ```

2. **Install dependencies:**
   ```bash
   # Install server dependencies
   cd magic_recipe_server
   dart pub get

   # Install client dependencies
   cd ../magic_recipe_client
   dart pub get

   # Install Flutter app dependencies
   cd ../magic_recipe_flutter
   flutter pub get
   ```

3. **Set up API keys:**
   - Obtain a Google Gemini API key from [Google AI Studio](https://makersuite.google.com/app/apikey)
   - Update the `geminiApiKey` in `magic_recipe_server/config/passwords.yaml`

4. **Start the database services:**
   ```bash
   cd magic_recipe_server
   docker compose up -d
   ```

5. **Generate Serverpod code:**
   ```bash
   cd magic_recipe_server
   serverpod generate
   ```

## Running the Application

### Development Mode

1. **Start the Serverpod server:**
   ```bash
   cd magic_recipe_server
   dart bin/main.dart --apply-migrations
   ```

2. **Run the Flutter app:**
   ```bash
   cd magic_recipe_flutter
   flutter run
   ```

### Web Version

1. **Build the Flutter web app:**
   ```bash
   cd magic_recipe_server
   serverpod flutter_build
   ```

2. **Start the server:**
   ```bash
   dart bin/main.dart
   ```

3. **Access the web app at:** `http://localhost:8080/app`

## API Endpoints

The server provides the following endpoints:

- `generateRecipe(ingredients: String)`: Generates a new recipe using AI
- `getRecipes()`: Retrieves all saved recipes
- `deleteRecipe(recipe: Recipe)`: Deletes a recipe from the database

## Configuration

### Database

The application uses PostgreSQL with the pgvector extension for potential future AI features. Database credentials are configured in `config/passwords.yaml`.

### Authentication

Email/password authentication is handled by Serverpod Auth. Email verification codes are logged to the console in development mode.

### AI Integration

Recipes are generated using Google's Gemini AI. The API key must be configured in the server passwords file.
